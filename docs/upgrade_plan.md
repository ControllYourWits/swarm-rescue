# 技术升级方案 — 基于 RoboMaster 顶级团队设计模式

> 参考: PolarBear 哨兵导航 (pb2025_sentry_nav), rm_vision 视觉框架, RoboRTS 架构

---

## 一、硬件升级清单

### 1. Scout (探测先遣机) — 核心升级

| 部件 | 当前 | 升级为 | 理由 |
|------|------|--------|------|
| **激光雷达** | RPLidar A1 (2D, 12m) | **Livox Mid-360** (3D, 40m, 非重复扫描) | RoboMaster 哨兵标配; 3D 点云可做地形分析, 检测废墟高低差 |
| **主控** | RK3588 | RK3588 (不变) | 8 核 ARM + 6 TOPS NPU, 够用 |
| **IMU** | BMI088 (板载) | **BMI088 + 外置 VG1000** (战术级) | 双 IMU 冗余, 提升 Point-LIO 精度 |
| **深度相机** | 无 | **Intel RealSense D455** (深度+RGB) | 近距障碍物 3D 感知, 废墟缝隙检测 |
| **计算扩展** | 无 | **M.2 NVMe SSD 256GB** | 存储地图 PCD 文件和 ONNX 模型 |
| **UWB 模块** | 预留 | **DWM3000 × 3** (每机一个) | 三机相对定位 ±10cm, 不依赖 GPS |

**Scout 传感器布局 (俯视图)**:
```
          前 (探路方向)
     ┌─────────────────┐
     │  [Livox Mid360] │  ← 顶部中心, 倾斜 15° 安装
     │   [D455 深度]    │  ← 前方, 朝前 15° 下倾
     │                  │
     │   [RK3588 主控]  │
     │   [BMI088 IMU]   │
     │   [DWM3000 UWB]  │
     │                  │
     │  [M3508×4 麦轮]  │
     └─────────────────┘
          后
```

### 2. Carrier (补给运输机) — 稳定性升级

| 部件 | 当前 | 升级为 | 理由 |
|------|------|--------|------|
| **主控** | 树莓派 4B | **RK3588s** (或 Orange Pi 5 Plus) | RPi4 算力不足, 无法跑 Nav2 全栈 |
| **激光雷达** | 无 | **RPLidar S2E** (2D, 30m, 防尘) | 跟随避障需要 |
| **4G 路由** | GL.iNet GL-X3000 | 不变 | 够用 |
| **电池** | 6S 22.2V | **6S 25.2V Li-Ion** (更高能量密度) | 增加续航 |

### 3. Specialist (作业处置机) — 感知升级

| 部件 | 当前 | 升级为 | 理由 |
|------|------|--------|------|
| **热成像** | 模拟 (仿真) | **FLIR Lepton 3.5** (160×120, 热成像) | 真实热成像, 烟雾/黑暗中检测幸存者 |
| **RGB 相机** | 无 | **海康威视 MV-CS060-10GC** (工业相机) | rm_vision 框架标配, 用于目标识别 |
| **机械臂** | 4-DOF 舵机 | **6-DOF 步进电机** + 力矩传感器 | 更多自由度, 力控防止二次伤害 |

## 二、软件架构升级

### 升级 1: 地形分析 (参考 PolarBear terrain_analysis)

**硬件需求**: Livox Mid-360 (Scout) 或任何 3D LiDAR

**功能**: 将 LiDAR 点云分类为:
- 可通行区域 (平坦, 坡度 < 15°)
- 障碍物 (可绕行)
- 不可通行 (悬崖/深坑/过高废墟)

```
LiDAR 点云
    ↓
terrain_analysis.py
    ├── 地面分割 (RANSAC 拟合地面平面)
    ├── 坡度计算 (法向量角度)
    ├── 高度差检测 (格栅内最大-最小高度)
    └── 输出: /scout/terrain_map (OccupancyGrid)
              /scout/traversable_cloud (PointCloud2)
    ↓
Nav2 Costmap Layer (自定义插件)
```

### 升级 2: Nav2 自定义控制器 (参考 PolarBear pb_omni_pid_pursuit)

**硬件需求**: 无 (纯软件)

**功能**: 针对麦轮全向底盘的路径追踪控制器, 替代默认的 DWB 控制器

改进点:
- 全向运动: 支持 vx, vy, wz 三轴独立控制
- 废墟避障: 动态调整路径跟踪速度
- 转弯平滑: 角速度前馈 + PID 反馈

### 升级 3: 生命周期管理 (参考 ROS2 best practice)

**硬件需求**: 无 (纯软件)

**功能**: 所有关键节点使用 ROS2 Lifecycle (LifecycleNode)

```
Unconfigured → [configure] → Inactive → [activate] → Active
     ↑                                                    |
     └── [cleanup] ← Inactive ← [deactivate] ←──────────┘
```

好处:
- 节点启动失败可重试, 不需要重启整个系统
- 可以远程暂停/恢复单个节点
- 状态可监控, 故障诊断更清晰

### 升级 4: 视觉流水线 (参考 rm_vision)

**硬件需求**: 海康威视工业相机 + FLIR Lepton 热成像

```
[海康相机] → /camera/image_raw
                 ↓
         armor_detector (YOLOv8-nano, NPU 推理)
                 ↓
         /detection/armors (目标像素坐标)
                 ↓
         coordinate_transformer (PnP 解算 → 3D 坐标)
                 ↓
         /detection/target_3d (目标在 base_link 下的 3D 位置)
                 ↓
         火控/导航决策
```

## 三、实施优先级

| 优先级 | 内容 | 需要硬件 | 工作量 |
|--------|------|----------|--------|
| **P0** | 地形分析节点 | Livox Mid-360 | 2 天 |
| **P0** | Nav2 自定义控制器插件 | 无 | 2 天 |
| **P1** | 生命周期管理 | 无 | 1 天 |
| **P1** | UWB 三机相对定位 | DWM3000 × 3 | 3 天 |
| **P2** | 视觉流水线 (YOLO + 热成像) | 工业相机 + Lepton | 5 天 |
| **P2** | 6-DOF 机械臂升级 | 步进电机 + 力矩传感器 | 5 天 |
